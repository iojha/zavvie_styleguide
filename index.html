Typography

General Guidelines

  • All fonts are in em. They size relative to the $base-font-size listed in body.
  • For media queries, just change the $base-font-size and all the font sizes should change accordingly.
  • Don't use $navy for regular text. Navy is for all linked font, and will confuse the user if used in non-linked text.

  • These banners are made using the following Sass mixin in the first code box below.
  • Use the class h1-lime-banner, in the markup. Replace "lime" with aqua for the aqua background color.


@mixin banner($color, $background, $fontsize, $padding, $margin-left, $margin-right) {
  color: $color;
  background: $background;
  padding: $padding;
  margin-left: $margin-left;
  margin-right: $margin-right;
}
                    

.h1-lime-banner {
  @include banner(#fff, $lime, h1, $banner-text-padding, 0, 0);
}

Buttons and links

General Guidelines

  • Buttons: Use for cases where user actions affect the website’s back-end or front-end. For example -- sign ups, creating new posts, making comments.
    • All buttons darken by 10% on hover.
    • Use coral for all primary actions such as submits.
    • Use aqua for all secondary actions.
    • Use lime button for all tertiary actions (only if coral and aqua have already been used)
  • Links (hrefs): Use for cases where user actions won’t affect the website at all --when the user is just being moved around in the site, where they are basically just “clicking and viewing”.
  • All buttons darken by 10% on hover.
  • Use coral for all submits.
  • Use aqua for all others.
  • Use lime button for ??
